# EVEE Design Document

Design decisions for the Evo Variant Effect Explorer (EVEE) webapp. This document captures the intent behind every card, field placement, color choice, and interaction so the UI can be faithfully replicated after refactoring.

## App Identity

- **Name**: EVEE (Evo Variant Effect Explorer)
- **Title bar**: `EVEE  Evo Variant Effect Explorer` (h1, 20px bold + 11px muted subtitle with 8px left margin)
- **Branding**: Goodfire logo (24px height) + Mayo Clinic logo (48px height) in header right
- **Color palette**: warm neutrals (`--bg: #faf8f5`, `--bg-card: #fff`, `--accent: #DB8A48`)

## Landing Page

- **Project description** centered above UMAP: 24px bold "EVEE" heading, 14px body text describing the project scope (4M+ ClinVar variants, Evo2 covariance probes, 0.97 AUROC), "Developed by" with both logos
- **UMAP scatter**: 30K random variants, two toggle modes:
  - *Pathogenicity*: blue-to-red gradient by predicted score
  - *ClinVar label*: categorical colors (pathogenic red, benign blue, VUS gray at 40% opacity)
- **Draw order**: shuffled randomly so no label/score systematically covers others
- Click any point to navigate to its variant card

## Variant Summary Card (top card)

Three-column flex layout (`align-items: flex-start`):

### Left: Variant Info
- **Line 1** (verdict-gene, 28px bold): `GENE  exon X/Y | Domain | LOEUF: 0.XX (constraint note)`
  - Delimiter: ` | ` with `&nbsp;` padding
  - LOEUF interpretation: <0.35 "highly constrained", <0.6 "constrained", <1.0 "tolerant", else "unconstrained"
  - Domains: filtered to exclude PDB-ENSP_mappings, AFDB-ENSP_mappings, ENSP_mappings, Gene3D; take first comma-separated ID if no name
- **Line 2** (verdict-id, 12px mono): `chrX:POS REF>ALT | c.HGVS | p.HGVS | Consequence`
  - Consequence moved to end (after protein change)
  - No gene strand display (redundant with c. notation)
- **Line 3** (verdict-meta): ClinVar label badge + stars + ACMG codes + disease + origin + gnomAD AF
  - gnomAD AF: "Not observed" when null/0, no "Absent from gnomAD" badge (redundant)
  - Frequency badges: Rare/PM2 (green), Polymorphism/BS1 (yellow), Common/BA1 (red)
- **Review line**: review status, submitter count, last evaluated date
- **External links**: ClinVar, gnomAD, dbSNP, UCSC, UniProt, Ensembl, OMIM, GeneCards (removed: VarSome, LitVar)

### Right: Predicted Pathogenicity
- **Label**: "PREDICTED PATHOGENICITY" (13px uppercase, black, two lines with `<br>`)
- **Score**: 48px monospace bold, colored by `scoreColor()` (blue gradient at low, red at high)

## Variant Interpretation Card

Loaded on-demand via `fetchAnalysis()` → `renderInterpretation()`. Structure matches PR #8's polling-based `fetchAnalysis()` + extracted `renderInterpretation()` for merge compatibility.

- **Section title**: "Variant Interpretation" + confidence badge (high/medium/low with colored background)
- **Mechanism**: 15px bold, one sentence
- **Summary**: 11px, 3-5 sentences, left-aligned
- **Key Evidence**: bulleted list of most important signals
- **Attribution**: "Generated by Claude" (10px muted)

## Top Disruptions Card

Contains both "Top Disruptions" (attribution-ranked) and expandable "All Disruptions" + "Variant Effects".

### Legend (two lines, block display)
- Line 1: `Left: ref (faded) / var (solid) | Color = likelihood ratio: pathogenic > leaning path. > neutral > leaning benign > benign`
- Line 2: `Right: delta (var - ref) | decreased (-) increased (+)`

### Disruption Row Layout
Uses unified `disruptionRow()` function for both Top and All sections:
- **Label** (140px): display name + info icon (i) + eval badge
- **Score bars** (flex:1): ref bar (7px, faded 50% opacity, top) + var bar (7px, solid, bottom) in container with `#f0ede8` background track
  - Bar color: `headColor()` likelihood ratio — 5-tier from blue (#27a benign) through gray (#bbb neutral) to red (#c55 pathogenic)
  - Zero values: no bar shown (background track visible)
- **Delta bar** (flex:1): centered at 50%, grows left (negative) or right (positive)
  - Color: `#ccc` gray when z < 1σ, `#c55` red when negative, `#4a9` green when positive (no alpha tiers)
  - Width: `z / 4 * 50%` of half-container (4σ = full half)
- **Delta value** (profile-value): signed decimal with +/- prefix, colored same as bar
- **Sigma label** (40px): `+X.Xσ` or `−X.Xσ` (minus sign U+2212 for negative)

### Top vs All
- **Top Disruptions**: uses attribution z-scores (pre-computed, passed as `it.z`)
- **All Disruptions**: computes z from `head_stats` client-side, grouped by category (Protein, Structure, Regulatory, Domains, Region), sorted by |z| descending
- Toggle: "Show all disruptions" / "Hide all disruptions"

### Histogram Popout
Click any row → expandable detail panel with:
- **Disruption heads**: dual side-by-side charts
  - Left: "Score Distribution (··· ref, --- var)" — ref score histogram with dotted ref line + dashed red var line
  - Right: "Delta Distribution (var − ref)" — delta histogram with dashed marker
  - Both with subtle `#f5f2ee` background, aligned to bar widths (`margin-left:140px; margin-right:90px`)
- **Effect heads**: single histogram with dashed marker
- **Adaptive binning**: per-head ranges from 0.5th-99.5th percentile (not fixed [-1,1] or [0,1])
- Charts use `createHistogramChart()` shared helper; benign = blue 35% opacity, pathogenic = red 35% opacity

## Computational Predictors Card

- **Legend**: likelihood ratio color legend (same 5-tier as disruption bars)
- **Predictors** (ordered): PhyloP 100-way, PhastCons 100-way, GERP, CADD, AlphaMissense, REVEL, 1−SIFT, PolyPhen-2, EVE, SpliceAI
- **SIFT**: displayed as `1 - value` so higher = more damaging, consistent with others; name shown as "1 − SIFT"
- **Bar**: 5-tier color via `tierColor()`: ≥0.8 red, ≥0.6 orange, ≥0.4 gray, ≥0.2 light blue, <0.2 blue
- **Value**: left-aligned, with superscript `*` when no database value (Evo2 prediction)
- **Footnote**: `* predicted by Evo2 (no database value available)` when any predictions present
- **Consensus line**: "X/N deleterious (supports PP3/BP4)"

## Nearest Neighbors Card

- **Summary**: "X pathogenic, Y benign, Z VUS among N closest"
- **VUS callout** (if variant is VUS): "This variant is unclassified (VUS). Its nearest neighbors suggest predominantly pathogenic/benign/mixed character."
- **Table columns**: Variant (truncated ID), Gene, Consequence, Label (badge), Score (%), Similarity (%)
- Click any row to navigate to that variant's card

## Score Distribution Card

- Full-width histogram, 80 bins, benign (blue 40%) vs pathogenic (red 40%)
- Vertical black line at this variant's score with "This variant" label
- X-axis: 0-100% with tick labels; Y-axis: count

## Tutorial Help System

- **Button**: `?` circle (22px, top-right of every card, absolutely positioned)
- **Panel**: grid animation (0fr → 1fr), 11px text, muted color, border-top separator
- Each card has unique help text (stored inline in `cardHelp(id, text)` calls)
- Help text uses `<b>` for field names, `&rsquo;` for apostrophes (not escaped quotes, which break template literals)

## Shared Helpers

- `extractDelta(rawD)` → `{ref, var, delta}` from raw disruption value (array or scalar)
- `deltaColor(z, sign)` → `{text, bar}` colors for z-score magnitude
- `createHistogramChart(canvasOrId, data, plugins)` → shared Chart.js histogram
- `cardHelp(id, text)` → help button + expandable panel HTML

## Section Title Styling

- 12px uppercase, 0.8px letter-spacing, **black** (`var(--text)`), 600 weight
- Category group titles: 11px uppercase, accent color, with bottom border

## Build Pipeline Notes

- `prebin(values, n_bins, lo, hi)`: adaptive range histogram binning
- VUS label filter: case-insensitive (`str.to_lowercase() == "vus"`)
- Dev mode (`--dev N`): allows `--umap` and `--neighbors`; uses `df_full` for neighbor/UMAP metadata
- Logos copied from `logos/` directory to staging
- Head quality filtering: AUPRC ≥ 0.7, MCC ≥ 0.3, Spearman ≥ 0.4
- Tissue-specific heads removed (chipseq/atacseq/chromhmm with tissue names)
- Prefix deduplication in attribution (one head per group)

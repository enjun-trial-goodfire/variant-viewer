# Variant Effect Viewer

Interactive browser for Evo2 ClinVar probe predictions. Visualizes pathogenicity
scores, disruption/effect profiles, UMAP embeddings, nearest neighbors, and VEP
annotations for ~230K ClinVar variants.

Extracted from [gfm_gen](https://github.com/goodfire-ai/gfm_gen) — the research
repo handles harvesting, training, and annotation. This repo handles scoring,
building, and serving the viewer.

## Quick start

```bash
uv sync
bash preflight.sh                    # check all data is present
uv run python build.py               # builds static site (~2 min on H200)
```

### Static serving (no interpretation)
```bash
python -m http.server -d webapp/build/ 8080
```

### With on-demand AI interpretation
```bash
ANTHROPIC_API_KEY=sk-ant-... uv run --extra serve python serve.py --build-dir webapp/build --port 8501
```

Interpretations are generated on-demand when you click a variant (~20s, cached to disk).
For public access on the cluster, tunnel the port:

```bash
source /mnt/polished-lake/scripts/config.sh
$SCRIPTS_DIR/bin/tunnel-url 8501
# → http://100.109.239.81:10017/ (expires in 24h)
```

## Data setup

The build requires **local data files** and **shared probe artifacts**.

**Local data** (`data/` directory, not in git):
```
data/
├── clinvar/
│   ├── deconfounded-full/
│   │   ├── metadata.feather         # ClinVar labeled variant metadata
│   │   └── annotations_v8.feather   # Ground truth annotation labels
│   ├── vus/
│   │   └── metadata.feather         # VUS variant metadata
│   └── submissions.feather          # [optional] ClinVar submission details
└── gencode/
    └── genes.feather                # GENCODE gene coordinates
```

These are generated by `gfm_gen`'s dataset pipeline. On the cluster, symlink:
```bash
ln -s /mnt/polished-lake/home/thomasd/projects/gfm_gen/data data
```

**Shared artifacts** (on cluster filesystem):
```
/mnt/polished-lake/artifacts/.../clinvar_evo2_deconfounded_full/
└── probe_v8/
    ├── scores.feather     # 3-view scores (ref_score_*, var_score_*, score_*)
    ├── embeddings/        # Covariance embeddings (for UMAP + neighbors)
    ├── config.json        # Head definitions (disruption_heads, effect_heads)
    ├── eval.json          # [optional] Per-head eval metrics
    └── attribution.json   # [optional] Ridge attribution model
```

Run `bash preflight.sh probe_v8` to validate everything exists.

## Pipeline

```
[gfm_gen]  harvest → train
               ↓
[this repo]  extract → finalize → eval → build → serve
```

Run the full chain:
```bash
bash pipeline/pipeline.sh /path/to/clinvar_evo2_deconfounded_full/probe_v8
```

Or step by step:
```bash
# 1. Score (8 GPU shards)
sbatch --array=0-7 pipeline/extract.sh --probe $ACTS/probe_v8 --activations $ACTS

# 2. Merge shards
sbatch --dependency=afterok:$JOBID pipeline/finalize.sh $ACTS/probe_v8

# 3. Build static site
uv run python build.py
```

## Architecture

| File | Role |
|------|------|
| `build.py` | Static site generator: scores + metadata → UMAP + neighbors + per-variant JSONs |
| `display.py` | Head display names and category grouping |
| `attribution.py` | Ridge regression attribution model (optional) |
| `index.html` | Single-page frontend (vanilla JS + CSS, no build step) |
| `serve.py` | Development server with live Claude interpretation |
| `probe/covariance.py` | MultiHeadCovProbeV2 (probe architecture + checkpoint loading) |
| `probe/binning.py` | Soft-binning for continuous probe heads |
| `pipeline/extract.py` | 3-view scoring: streams activations → embeddings + scores |
| `pipeline/finalize.sh` | Merge shard scores + index embeddings |
| `pipeline/pipeline.sh` | End-to-end chain (extract → finalize → eval → build) |
| `pipeline/interpret.py` | Claude API variant interpretation |
| `pipeline/serve.sh` | SLURM: build + serve in one job |
| `clinvar_submissions.py` | Parse ClinVar XML → submissions.feather (run once) |
| `preflight.sh` | Validate all inputs exist before building |
| `SCHEMA.md` | JSON data contract between build.py and index.html |

## Probe heads

The probe has two types of heads (defined in `config.json`):

- **Disruption heads** (~500): Scored on reference and variant views separately.
  The delta (var − ref) shows what the mutation disrupted. Examples: helix structure,
  domain membership, ChIP-seq peaks, conservation.

- **Effect heads** (~200): Scored on the diff view (var − ref activations).
  Predict variant-level properties directly. Examples: pathogenicity, CADD,
  AlphaMissense, SpliceAI, Pfam domain membership.

Config keys: `disruption_heads` and `effect_heads` (older probes use `ref_heads` / `diff_heads`).

## Dependencies

- `goodfire-core >=0.3.0` — activation storage, streaming, covariance probe
- `polars` — data manipulation (not pandas)
- `torch` — probe inference, GPU similarity
- `umap-learn` + `scikit-learn` — UMAP embedding
- `orjson` — fast JSON serialization (232K variant files)
- `anthropic` — [optional] Claude API for live interpretation
